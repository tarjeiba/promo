Eksempelrapport: Primtallsfaktorisering

Bakgrunn

Ifølge aritmetikkens fundamentalteorem kan ethvert heltall større enn 1 skrives som et produkt av primtall på én og kun én måte. For eksempel er \(28 = 2^{2}\cdot 7\) og \(11 = 11\).

Jeg skal nå forsøke å løse Project Euler oppgave 3, som spør om følgende:

Primtallsfaktorene til 13195 er 5, 7, 13, og 29.

Hva er den største primtallsfaktoren til tallet 600851475143?

Algoritme

Merk at oppgaven kun er ute etter den største primtallsfaktoren. Siden det kun er én måte å faktorisere et tall til sine primtallsfaktorer på, kan vi jobbe med mindre og mindre tall ved å «dele bort» faktorene når vi finner dem.

Vi har funnet delt bort alle primtallsfaktorer når vi sitter igjen med tallet 1.

For eksempel, dersom vi ønsker å finne den største primtallsfaktoren 28, kan vi gjøre dette ved å gjøre følgende:

  • Starte med å teste faktoren 2
  • Dersom 2 er en faktor, dele bort 2-ere så lenge det er mulig \(28/2 = 14\), \(14/2 = 7\), når vi kommer til 7 er det ikke mulig å dele bort flere 2-ere.
  • Den største primtallsfaktoren i 28 er altså enten 7, eller den største primtallsfaktoren i 7.
  • Vi tester den neste faktoren (\(2+1 = 3\)).
  • \(7/3\) går ikke opp
  • Vi tester den neste faktoren, 4.
  • Går ikke opp.
  • Vi tester den neste faktoren, 5.
  • Går ikke opp.
  • Vi tester den neste faktoren, 6.
  • Går ikke opp.
  • Vi tester den neste faktoren, 7.
  • \(7/7=1\), vi har kommet til 1, og delt bort alle faktorer, altså var 7 den største primtallsfaktoren.

Vi kan oversette dette til omtrentlig følgende pseudokode: 

hent inn TALL
sett TESTFAKTOR til 1

så lenge TALL er større enn 1, gjør:
    øk TESTFAKTOR med 1
        så lenge TESTFAKTOR er en faktor i TALL, gjør:
            del bort TESTFAKTOR fra TALL

Etter at dette programmet har kjørt, vil TALL være 1 og TESTFAKTOR vil holde den største primtallsfaktoren i det opprinnelige TALL.

Program

import time

starttid = time.time()

tall = 600851475143
testfaktor = 1

print(f"Starter med tallet {tall}.")

while tall != 1:
    testfaktor += 1
    while tall % testfaktor == 0:
        print(f"... deler bort {testfaktor}.")
        tall //= testfaktor

sluttid = time.time()

print(f"Den største primtallsfaktoren i {TALL} er {testfaktor}.")
print(f"Denne utregningen tok {sluttid-starttid:.4f} sekunder.")

Starter med tallet 600851475143.
... deler bort 71.
... deler bort 839.
... deler bort 1471.
... deler bort 6857.
Den største primtallsfaktoren i 600851475143 er 6857.
Denne utregningen tok 0.0044 sekunder.

Kommentarer

Programmet har mange mulige optimaliseringerer. Noen eksempler:

  1. Det sjekker alle partall, selv etter den har delt bort alle toere.
  2. Det sjekker alle verdier opptil TALL, selv om TALL må være et primtall dersom alle verdier opptil sqrt(TALL) har blitt sjekket.

Disse forbedringene ville kunne endret hvor lang tid programmet bruker på å kjøre.

Feilhåndtering

Programmet over håndterer feil fra spilleren på en lite grasiøs måte. Spillet tvinger seg videre, uten å la spilleren forsøke å taste inn på nytt.

Starten på en forbedring vil være å restrukturere koden slik at teksten, selve «historien» vår, kommer foran. Vi kan da forsøke å skille mellom historien, og selve spillmotoren.

Når det er gjort kan vi så sette opp spillet på tilsvarende måte som siste, men vi begynner å ta høyde for feil fra brukeren. Legg også merke til at vi nå «pynter» på brukerens input ved først å gjøre alle bokstavene til små bokstaver ("TestStreng".lower() => "teststreng") og ved å fjerne eventuelle mellomrom og tomme tegn på starten og slutten av strengen (" test ".strip() => "test").

Vi kunne da ha «pakket» hver input inn med f-strenger avsluttet med lower og strip, noe à la 

valg = input(f"'{valg_a}' / '{valg_b}'").lower().strip()

men dette ville ikke hjulpet oss med å be brukeren om å taste inn på nytt dersom det ikke fungerte.

Vi kan heller pakke det inn i en funksjon, hvor vi venter til brukeren har tastet et gyldig alternativ ved å bruke en while-løkke. 

def be_om_input(alt1, alt2):
    """Ber spilleren om å taste inn input til spillet, og gir seg først når
    spilleren har tastet inn noe gyldig.
    
    Returnerer valget brukeren tastet inn når dette er gyldig.

    Eksempel på bruk:
    valg = be_om_input('gå', 'løp')

    'gå' / 'løp' >>> går
    Ugyldig alternativ, prøv igjen.
    'gå' / 'løp' >>> gå     # valg er nå 'gå'
    """
    valgt = None
    while valgt != alt1 and valgt != alt2:
        valgt = input(f"'{alt1}' / '{alt2}'").lower().strip()
    return valgt

Fiendekamp med stein-saks-papir

La oss begynne med å lage oppsettet for en stein-saks-papir-kamp, før vi i det hele tatt legger det til i eventyret vårt.

Jeg har valgt å gjøre det her via to funksjoner, avgjoer som tar inn to valg og avgjør om det var det første eller andre valget som vant, samt spill_steinsakspapir som kjører en kamp med stein-saks-papir til enten spilleren eller maskinen har vunnet. 

import random as rnd

def avgjoer(vaapen1, vaapen2):
    """Tar inn to strenger, hver enten 'stein', 'saks', eller 'papir'.

    Returverdier er som følger:
    vaapen1 > vaapen2  => 1
    vaapen2 > vaapen1  => 2
    vaapen1 == vaapen2 => 0

    Rangeringen er
    'stein' > 'saks' > 'papir'
    og 'papir' > 'stein'

    """

    if vaapen1 == "stein":
        if vaapen2 == "stein":
            return 0
        elif vaapen2 == "saks":
            return 1
        elif vaapen2 == "papir":
            return 2
    elif vaapen1 == "saks":
        if vaapen2 == "stein":
            return 2
        elif vaapen2 == "saks":
            return 0
        elif vaapen2 == "papir":
            return 1
    elif vaapen1 == "papir":
        if vaapen2 == "stein":
            return 1
        elif vaapen2 == "saks":
            return 2
        elif vaapen2 == "papir":
            return 0
    else:
        print("Det skjedde noe feil i avgjørelsen.")
            
def spill_steinsakspapir():
    """En funksjon du kan spille stein-saks-papir mot. Den returner følgende:
    spill_steinsakspapir() => 1 # spilleren vant
    spill_steinsakspapir() => 2 # funksjonen vant
    
    Funksjonen fortsetter å spille til det er kåret en vinner.

    Rangeringen av 'stein', 'saks', og 'papir' gjøres i 'avgjoer'-funksjonen.
    """

    vaapen = ["stein", "saks", "papir"]
    while True:
        spillers_valg = input(f" '{vaapen[0]}' / '{vaapen[1]}' / '{vaapen[2]}' ")
        spillers_valg = spillers_valg.lower().strip()
        mitt_valg = rnd.choice(vaapen)
        avgjoerelse = avgjoer(spillers_valg, mitt_valg)
        if avgjoerelse == 0:
            print("Uavgjort, vi prøver igjen...")
        elif avgjoerelse == 1:
            print("Æsj, du vant.")
            return 1
        elif avgjoerelse == 2:
            print("Ha ha, jeg vant!")
            return 2
        else:
            print("Vi klarte visst ikke helt dette, du og jeg.")

import random as rnd

def be_om_input(alternativer):
    """Tar inn en liste med alternativer og gir spilleren mulighet til å velge
    en av dem. Returnerer indeksen i lista for valget brukeren tok. Gir også 
    brukeren mulighet til å velge ved å taste inn et tall.

    For eksempel:

    >>> svaridx = be_om_input(['stå opp', 'sov videre'])
    1) 'stå opp'
    2) 'sov videre'
    [1-2] / tekst: stå opp # svaridx er nå 0, siden 'stå opp' lå på indeks 0 i alternativer
    """
    valgt = None
    alternativstrenger = [f" {num+1}) '{alternativ}'" for
                          num, alternativ in enumerate(alternativer)]
    alternativstreng = "\n".join(alternativstrenger)
    alternativstreng += f"\n[1-{len(alternativer)}] / tekst: "
    idx = -1
    while idx == -1:
        valgt = input(alternativstreng + " \n").lower().strip()
        try:
            idx = int(valgt) - 1
            if idx in range(len(alternativer)):
                break
            else:
                idx = -1
                continue
        except ValueError:
            pass
        try:
            idx = alternativer.index(valgt)
        except ValueError:
            pass
    return idx

Battleships

Regler

Battleships spilles på et rutenett med ti ganger ti ruter, man navngir kolonnene A til J og radene 1 til 10. Rute "B8" viser da til rute nummer åtte ovenfra, nummer to fra venstre, som i et regneark. Hver spiller setter ut båter på sitt rutenett, hvor størrelsen sier hvor mange ruter båten tar.

Antall Type Størrelse
1 Hangarskip 5
1 Jagere 4
1 Fregatter 3
2 Korvetter 2
2 Undervanssbåt 1

To spillere plasserer ut båtene på hvert sitt brett. Spiller 1 begynner så å «skyte» på spiller 2s brett ved å si koordinater, for eksemepel "I5". Spiller 2 forteller så spiller 1 hvorvidt det var treff eller bom, og skyter så sitt eget skudd.

Din oppgave

Du skal lage en kunstig intelligens som spiller Battleships. Jeg kommer til å sette opp et «spilleder»-program, hvor dette kommer til å kjøre din kode opp mot en av medelevene dines.

Mitt program ser ut som følger:

def bokstav_til_kolonne(bokstav):
    """Tar inn en bokstavverdi, og returnerer hvilken kolonne det tilsvarer,
    null-indeksert.

    Eksempler:
    bokstav_til_kolonne('A') # => 0
    bokstav_til_kolonne('D') # => 3
    bokstav_til_kolonne('J') # => 9
    bokstav_til_kolonne('C') # => 2
    """
    return "ABCDEFGHIJ".index(bokstav.upper())

def finn_verdi(koordinatstreng):
    """Tar inn en koordinatstreng på formen "bokstavtall" og returnerer
    hvorvidt det er en båt i den koordinaten.
    """

    bokstav = koordinatstreng[0]
    rad = int(koordinatstreng[1])
    kol = bokstav_til_kolonne(bokstav)
    
    verdi = mitt_spillbrett[rad][kol]
    return verdi
